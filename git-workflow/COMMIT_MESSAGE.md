# コミットメッセージ規約

## 基本フォーマット
```
<type>: <subject>

<body>

<footer>
```

## Type（必須）

- `feat`: 新機能
- `fix`: バグ修正
- `docs`: ドキュメントのみの変更
- `style`: フォーマット（コード動作に影響なし）
- `refactor`: リファクタリング
- `test`: テスト追加・修正
- `chore`: ビルド、補助ツールの変更

## 例

### 良い例
```
feat: ユーザー認証機能を追加

- JWT認証を実装
- ログイン/ログアウトAPI追加
- セッション管理機能
```
```
fix: タスク削除時のエラーを修正

削除ボタンクリック時に404エラーが発生していた問題を解決
```

### 悪い例
```
update code
```
```
いろいろ修正
```

## Tips

- 1行目は50文字以内
- 現在形で書く（"追加した" ではなく "追加"）
- 何を変更したかより、なぜ変更したかを書く
